template<size_t StackSize = 4096U, size_t MaxStackAllocationSize = std::numeric_limits<size_t>::max(), size_t MaxHeapBlockSize = 65536U>
virtual HOT void do_deallocate ( void *  _p,
const size_t  _bytes,

)
inlineoverrideprotectedvirtual

Does nothing. Deallocation is a no-op in monotonic allocation schemes.

This method is overridden to comply with the std::pmr::memory_resource interface. As the monotonic_pool only supports monotonic allocation without individual deallocation, this implementation is a no-operation function. Any memory allocated by this resource will only be freed upon resetting or releasing the entire pool.

Parameters
[in]_pPointer to the memory block to be deallocated. Must not be null. Currently unused.
[in]_bytesSize of the memory block to be deallocated, in bytes. Currently unused.
[in]_alignmentAlignment constraint for the start of the allocated memory block. Must be a power of two. Currently unused.
Postcondition
This function is a no-op and calling it does nothing, as the pool only supports deallocating all memory at once. Consider using reset() or release() instead.
